Why Framework ?

If you're making a really really simple site (think of the web back in 1992), you can just do it all with hard-coded HTML and some CSS. And if you want to make a modern Web Application, you don't actually need to use a framework for that, either, You can use jsp /servlet standard style You can instead choose to write all of the logic you need yourself, every time. You can write your own data-persistence/storage layer, or custom SQL for every single database access. Or own authentication and session handling layers or your own template rendering logic. or your own exception-handling logic. or your own security functions. Or your own unit test framework to make sure it all works fine. or your own â€¦. Then again, if you do use a framework, you'll be able to benefit from usually peer-reviewed and very well tested work of dozens if not hundreds of other developers, who may well be better than you. You'll get to build what you want rapidly, without having to spend time building or worrying too much about the infrastructure items listed above.
